Technical Animation Guidelines

Overview

UI components and pages may require animations to enhance the user experience. The goal of this document is to provide technical direction for animating components in an easy to use and consistent manner while also maintaining optimal webpage performance.

Triggers

  1. page load - an animation runs when the page loads
  2. user action - an animation runs when the user interacts with the page or a component
    1. scroll - when the user scrolls up/down the page, a component animation runs (also known as "lazy load" or animate on scroll)
    2. click/touch - when the user clicks/touches an element, an animation runs
    3. while scroll and click/touch are the more common use cases, there are other user events such as: focus, input, mouse, drag, touch, etc. that may require animations
      1. available event types: https://developer.mozilla.org/en-US/docs/Web/API/Event

Strategy

In order to achieve the expected animation, one or more of the following should be used in this order:

  1. AOS.js library, version 2.3.4 - documentation, demo
  2. Custom CSS animations or JS Animation API
  3. Custom JS animations
  4. Custom animation libraries

Functional Usage

Using the AOS.js Library on components in the CMS

  • Animating content within a Content Block Gravity can be done by web producers
  • Animating widgets in the CMS does require minor development work as the front-end widget code will need to be updated, the benefit is all widgets on the site now receive the animation

http://uat-dcm.equinix.com/testing/pete/aos/v2.3.4)

Multiple Components

Example: a Section Wrapper with multiple Content Blocks

  1. add a unique ID to the Section Wrapper to serve as the AOS anchor
  2. for each component (or element), add the following data attributes:
    1. data-aos="<transition>", e.g. fade-up
    2. data-aos-delay="<value>", e.g. 150, 250, 350, etc.
    3. data-aos-anchor="<unique_id>", unique_id = ID of the Section Wrapper (or wrapping element)

Example: Hero Banner with heading, supporting text, and CTA button

  1. add the data attribute to the element/component to be animated
    1. data-aos="<transition>", e.g. fade-up
  2. for each addition component (or element), add the following data attributes:
    1. data-aos="<transition>", e.g. fade-up
    2. data-aos-delay="<value>", e.g. 150, 300, etc.
    3. data-aos-anchor="<unique_id>", unique_id = ID of the Section Wrapper (or wrapping element)

Single Component

  1. add the data attribute to the element/component to be animated
    1. data-aos="<transition>", e.g. fade-up
