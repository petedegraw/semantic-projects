DX File Name Structure

Project-discipline-documenttype-documenttype2-version-date

Where:

  • Project

    • 4 letter or less (Platform Equinix = PE)
    • or 4-4 as required (Partners Phase 2 = PAR-P2)

  • Discipline

    • Designated by UX/UI/CON/PM

  • Document Type

    • Full name for clarity, no spaces or dashes (ContentStrategyDeck)
    • Use multiple sections if needed (Mockup-Veterans, Mockup-Internships)

  • Version

    • Optional per document type

  • Date

    • Appended only when a document is complete

    • Appended only to documents to be shared

Governance

If a file does not have a date it is NOT to be distributed outside DX

Examples

Ins-UI-Mockup-Landing (for Insights Landing Page Mockup)

Ins-UX-DocumentPackage-V3-14May2020 (for Insights Document Package V3)

Par-P2-CON-ContentStrategyDeck-08May2020 (for Partners Phase 2 Content Strategy)

Notes

Goals

  • Naming structure that is easy for DMX,PM, and Prod teams to create, understand, and locate.
  • Naming that makes some intuitive sense to external teams, including Prod (secondary)
  • Consistency
  • Clarity on what is complete, latest, ready to share, or in process

Best Practices Employed

  • No special characters *(^&%
  • No use of “final”
  • Use dash instead of space or underscore
  • Keep it short
  • Consistency of case (Title, all lowercase, all upper, etc)
  • Full descriptions for clarity in type of document
  • Narrows from left to right in order of how we usually search and prioritize
